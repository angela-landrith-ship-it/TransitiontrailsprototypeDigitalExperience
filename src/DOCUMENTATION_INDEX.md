## üîÑ Documentation Standards

### Updating Documentation

When making changes to the codebase:

1. **Update relevant docs** in the same PR
2. **Keep examples current** with code changes
3. **Add screenshots** for UI changes
4. **Version documentation** for major releases

### Documentation Format

All documentation follows:
- **Markdown** format
- **GitHub Flavored Markdown** syntax
- **Semantic headings** (H1, H2, H3)
- **Code blocks** with syntax highlighting
- **Tables** for structured data
- **Links** for cross-references

### Version Control

Documentation is versioned alongside code:
- Major versions: Breaking changes
- Minor versions: New features
- Patch versions: Bug fixes and clarifications
